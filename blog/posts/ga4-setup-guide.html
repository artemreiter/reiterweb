1. GA4 Basics for E-commerce

Unlike Universal Analytics, GA4 uses an event-based model. Everything is an event: page views, button clicks, purchases, scrolls. This gives you much more flexibility but requires a different mindset.

Key Differences for E-commerce:

  • No more "sessions" as the primary metric
  • Enhanced measurement tracks scrolls, outbound clicks, site search automatically
  • User-centric reporting (tracks users across devices)
  • Built-in machine learning for predictive metrics
  • Better privacy controls for cookie-less tracking

2. Step-by-Step Setup Process

Step 1: Create Your GA4 Property

  1. Go to analytics.google.com
  2. Click "Create Property"
  3. Name: Your Store Name
  4. Reporting time zone: Your business timezone
  5. Currency: USD (or your store currency)

Step 2: Set Up Data Streams

For e-commerce, you need a Web data stream:

  1. Choose "Web" as your platform
  2. Enter your website URL
  3. Stream name: "Website"
  4. Enable "Enhanced measurement"

Step 3: Install the Tracking Code

Option A: Direct gtag.js installation:

<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>

  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>

Option B: Google Tag Manager (recommended for e-commerce)

3. Enhanced E-commerce Configuration

This is where most stores fail. You need to send specific events with the right parameters:

Required E-commerce Events:

  • view_item - Product page views
  • add_to_cart - Add to cart actions
  • begin_checkout - Checkout initiation
  • purchase - Completed transactions
  • view_cart - Cart page views

Purchase Event Example:

gtag('event', 'purchase', {
  transaction_id: 'T_12345',
  value: 25.42,
  tax: 4.90,
  shipping: 5.99,
  currency: 'USD',
  items: [{
    item_id: 'SKU_12345',
    item_name: 'Stan and Friends Tee',
    item_category: 'Apparel',
    price: 9.99,
    quantity: 1
  }]
});

4. Conversion Tracking Setup

In GA4, any event can be a conversion. Mark these as conversions:

  1. Go to Admin ‚Üí Events
  2. Toggle "Mark as conversion" for:
    • purchase
    • add_to_cart (optional)
    • begin_checkout (optional)

5. Testing & Validation

Before going live, verify everything works:

Testing Checklist:
  • Real-time reports show your activity
  • DebugView shows events firing correctly
  • Purchase events include transaction_id
  • Revenue matches your store backend
  • Product names appear correctly in reports

6. Common Mistakes to Avoid

‚ùå Mistake #1: Duplicate Tracking

Having both Universal Analytics and GA4 firing on the same events causes data inflation. Migrate completely or use separate testing environments.

‚ùå Mistake #2: Missing Transaction IDs

Without unique transaction IDs, GA4 can't deduplicate purchases. Always include transaction_id in purchase events.

‚ùå Mistake #3: Wrong Currency

Set currency at the property level AND in every purchase event. Mismatched currencies destroy revenue reporting.

‚ùå Mistake #4: Not Testing on Mobile

50%+ of e-commerce traffic is mobile. Always test your tracking on actual mobile devices, not just desktop browsers.
